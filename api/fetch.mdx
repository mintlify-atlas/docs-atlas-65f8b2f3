Ant provides a `fetch()` function for making HTTP requests, compatible with the browser Fetch API standard.

## fetch()

Makes an HTTP request and returns a Promise that resolves to a Response object.

<ParamField path="url" type="string" required>
  The URL to fetch
</ParamField>

<ParamField path="options" type="object">
  Optional request configuration
  
  <ParamField path="method" type="string" default="GET">
    HTTP method: `"GET"`, `"POST"`, `"PUT"`, `"DELETE"`, etc.
  </ParamField>
  
  <ParamField path="headers" type="object">
    Request headers as key-value pairs
  </ParamField>
  
  <ParamField path="body" type="string">
    Request body (for POST, PUT, etc.)
  </ParamField>
</ParamField>

**Returns:** Promise that resolves to a Response object.

## Response Object

The Response object represents the response to a request.

### Properties

<ResponseField name="ok" type="boolean">
  True if status is in the range 200-299
</ResponseField>

<ResponseField name="status" type="number">
  HTTP status code (e.g., 200, 404, 500)
</ResponseField>

### Methods

#### response.text()

Returns a Promise that resolves to the response body as a string.

```javascript
const response = await fetch('https://example.com');
const text = await response.text();
console.log(text);
```

#### response.json()

Parses the response body as JSON and returns a Promise that resolves to the parsed object.

```javascript
const response = await fetch('https://api.example.com/data');
const data = await response.json();
console.log(data);
```

## Examples

### GET Request

```javascript
const fetchJson = (url, options) => fetch(url, options).then(r => r.json());

const { ip } = await fetchJson('https://ifconfig.co/json');
console.log(ip);
```

### POST Request

```javascript
const response = await fetch('https://httpbingo.org/post', {
  method: 'POST',
  body: JSON.stringify({ runtime: 'ant' }),
  headers: {
    'Content-Type': 'application/json',
    'User-Agent': 'ant/alpha (ant)'
  }
});

const { json, headers } = await response.json();
console.log(`${JSON.stringify(json)}\n${JSON.stringify(headers)}`);
```

### Error Handling

```javascript
try {
  const response = await fetch('https://api.example.com/data');
  
  if (!response.ok) {
    throw new Error(`HTTP error! status: ${response.status}`);
  }
  
  const data = await response.json();
  console.log(data);
} catch (error) {
  console.error('Fetch failed:', error);
}
```

### Custom Headers

```javascript
const response = await fetch('https://api.example.com/protected', {
  headers: {
    'Authorization': 'Bearer token123',
    'Accept': 'application/json'
  }
});

const data = await response.json();
```

### Multiple Requests

```javascript
const test_get = async () => {
  const { ip } = await fetchJson('https://ifconfig.co/json');
  console.log(ip);
};

const test_json = async () => {
  const test = await fetchJson('https://themackabu.dev/test.json');
  console.log(test);
};

const test_post = async () => {
  const { json, headers } = await fetchJson('https://httpbingo.org/post', {
    method: 'POST',
    body: JSON.stringify({ runtime: 'ant' }),
    headers: {
      'Content-Type': 'application/json',
      'User-Agent': 'ant/alpha (ant)'
    }
  });
  console.log(`${JSON.stringify(json)}\n${JSON.stringify(headers)}`);
};

Promise.all([test_get(), test_post(), test_json()]);
```

### PUT Request

```javascript
const response = await fetch('https://api.example.com/users/123', {
  method: 'PUT',
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  }),
  headers: {
    'Content-Type': 'application/json'
  }
});

if (response.ok) {
  console.log('User updated successfully');
}
```

### DELETE Request

```javascript
const response = await fetch('https://api.example.com/users/123', {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer token123'
  }
});

if (response.ok) {
  console.log('User deleted');
}
```

## Implementation Details

- Built on top of libuv and tlsuv for efficient async I/O
- Supports HTTP and HTTPS protocols
- Automatically sets `User-Agent` header to `ant/{version}`
- Response bodies are buffered in memory (initial capacity: 16KB, grows as needed)
- Fetch operations run as microtasks in the event loop
- TLS/SSL certificate verification is enabled by default

## Limitations

- Streaming response bodies is not yet supported
- Request/Response objects don't expose all standard Fetch API properties
- No built-in support for form data or file uploads
- Redirect handling is automatic (cannot be configured)
